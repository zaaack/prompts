## 1. Git 自动化流 (强制)

* **执行前**：运行 `git status`。若有未提交改动，立即执行：
`git add . && git commit -m "chore: save point before task"`
* **完成后**：任务结束立即执行：
`git add . && git commit -m "<type>: <description>"`
*(注：type 须符合规范，如 feat/fix/refactor/docs)*

## 2. 核心行为准则

* **先思后做**：动手前简述假设。如有歧义或更简方案，必须先询问，不要闷头做。
* **极简主义**：只写实现目标所需的最小代码。禁止过度封装、禁止预留“灵活性”、禁止实现未要求的功能。
* **外科手术式修改**：仅改动与任务直接相关的行。严禁随手重构邻近代码或调整格式。
* **自清现场**：仅清理因**本次修改**产生的废弃代码（如无用 import），不碰原有的陈旧代码。
* **目标导向**：复杂任务须先列出简要步骤：`1. [步骤] → 验证: [标准]`。

## 3. 输出要求

* 所有的 `git` 或 `bash` 命令直接调用工具执行，无需询问权限。
* 最终回复须简要提及已生成的 Commit 信息。
